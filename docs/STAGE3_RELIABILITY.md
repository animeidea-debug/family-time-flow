# Stage 3：运行可靠性基线

## 数据库安全

- 数据库默认位置为 `web/backend/family-time-flow/data/ftf.db`，可用 `DB_PATH` 覆盖。
- 每次写入先生成同目录临时文件，再通过原子重命名替换正式数据库，避免进程中断留下半个文件。
- 已存在的数据库会在服务启动、执行迁移之前复制到 `backups/`。
- 默认保留最近 7 份启动备份；可用 `BACKUP_DIR` 和 `BACKUP_LIMIT` 调整。
- 数据库损坏或格式无法读取时，服务会启动失败并保留原文件，绝不会静默创建空家庭。

## 恢复流程

1. 停止 FamilyTimeFlow 后端。
2. 复制当前 `ftf.db` 到独立位置，保留现场。
3. 从 `backups/` 选择启动前的最近备份。
4. 将备份复制为配置的 `DB_PATH`。
5. 启动后端并检查 `/api/health` 与家庭首页。

不要在服务运行期间直接覆盖数据库文件。

## 运行状态

`GET /api/health` 会返回版本及存储就绪状态。服务收到 `SIGTERM` 或 `SIGINT` 时会停止接收请求、关闭数据库并退出，适配容器正常重启。

## 当前边界

- 备份与数据库位于同一存储时，不能替代 NAS 快照或异机备份。
- 本阶段不修改 NAS 容器、卷或备份任务；这些应在 NAS 项目确认运行状态后协同配置。
- Immich 仍未接入，也不会在启动时自动探测。
